「Webコンテンツ作成」の成果

本テーマの学習目的

webページの作成方法の基本を6回の授業で学びながら、その成果をプロジェクト課題に反映していき、自分の力で簡易的なwebサイトを作る。
HTML,CSS,JavaScript,DOMについて理解し、さらにイベント処理について知ることで、一般的なwebサイトにある、何かしらのアクションを起こすことでページが切り替わる といった一般的なwebページでは当たり前にできることが、自分たちでもできるようになる。
つまり、最終的な目標としては、「簡易的なwebページを作成できるスキルを身につける。」ということである。

プロジェクト課題について

プロジェクト課題はグルメ情報を選択。このページは、まずユーザに知りたい飲食店のジャンルを選択してもらう。 ジャンルを選択し、検索ボタンを押したら、そのジャンルの近くの飲食店の情報ををサーバーから入手する。その情報を一覧としてwebページに表示させる。
また、何度でも同じページで再検索がかけられる状態にする。
ユーザが再び検索を実行したときには、まず前回入手してページ上に表示したデータを全て消去する。
そのあとは先ほどまでと同じく、サーバーから情報を入手して、その情報をwebページに表示させる。

第1回 WWW と HTML

今回は、HTMLの基本から学んだ。
HTMLは要素から作られ、要素には、ブロック要素とインライン要素の二種類がある。
要素に属性を追加することもできる。
HTMLは文書を作成することができるため、webサイトに文章を乗せることができる。
MDN Web Docsを活用して、要素の使い方を知ることができる。
ページソース、HTMLバリデーションを活用したHTMLのデバッグ方法を学んだ。

第2回 CSSによる WEB ページの装飾

今回は、CSSを使った、webサイトの見た目の装飾方法について学んだ。
HTMLのhead要素にlink要素を追加し、CSSファイルを対応させることで、 CSSに則った装飾がされるようになる。
CSSの記述方法は、(装飾する対象){装飾する内容}と書く。
CSSを使うことで、文字の色を変更できたり、背景色を変えたり、 webサイトのレイアウトを変えたりなどさまざまなことが可能になる。
web開発ツールを活用したデバッグ方法。その場で必要なCSSの修正をかけて試してみたり、修正が必要な箇所を簡単に見つけることができる。 こういった機能は、規模の大きいwebサイトを作るときには特に有用な機能なため、今のうちから使えるようになったいると、非常に便利で、効率よくwebサイト作成を進められる。

第3回 JavaScript の基本

今回はjavaScriptを学び始めた。プログラミングIにてJavaは学んでいるが、
名前が似ているだけで、JavaとJavaScriptは全くの別物ではあるが、似ている部分も一部ある
例えばfor文やif文はほぼ同じように書ける上に動作もかなり近い。
しかし、型の区別や配列の記述、オブジェクトなど大きく違う要素もある。
特に大きく違うのは、エラーが出ずに、意図しない動作をしてしまうことがJavaScriptではあり得ること。
そのため、Web開発ツールなどをうまく活用し、念入りにデバッグを行うことで、
意図しない動作を起こさないかを確認することがもっとも重要だ。

第4回 DOM と JavaScript

DOMの概念からDOMのプログラミングに関して学んだ。
DOMを利用することで、webサイトの記述を付け足したり逆に消したりといった、
要素の付け足しや削除を行える。だが、JavaScriptの記述を変えるだけで同じことが出来るため、このままなら実用性がない
だがDOMの利点もしっかりある。イベント処理を実装することで、ユーザ側の操作に応じて、webページを変えることができる。
そのためイベント処理を実装してからが。DOMの本領発揮と言える。
DOMプログラムは難しいため、コンソール機能から編集することが重要であり、さらにいうなら、
一気に作ろうとしない。少しずつ確かめて行ってから記述をして行くことがとても重要だと感じた。
本当に1行ずつ試す。くらいの方が安定する。逆に一気に作ると、その時の状況やエラーの原因を特定するのに時間がかかる。
結果的に一気に作ろうとして失敗し、地道にやるより時間がかかることの方が多いから、1ステップずつ手順を踏むことが重要だと認識した。

工夫点，苦心点，強調点，感想

プロジェクト課題ページについて、最初は店舗名も全てまとめてp要素として他と区別をしていない状態であった。
しかし、それだと、利用したユーザがどこからが別の店舗なのかが分かりにくいと感じ、分かりやすくするため、店舗名だけあえてh3要素に変えた。
さらに、店舗名だけを特別に見た目を変えることで、分かりやすくしたのはもちろん、強調、視線が吸い寄せられるようにできたと思う。
検索方法も、プルダウンメニューを使用することで、ユーザに分かりやすくでき、さらに自分がプログラムを記述する際にも、入力が制限されているおかげで、予期せぬ入力が起こり得ないため、記述もしやすかった。
また、予期せぬ入力が起こり得ないことで、バグの発生も抑制できる。
また、検索結果の最後を分かりやすくするために、検索結果の最後の記述も追加した。
inndex.htmlは、CSSでの装飾はしっかり時間をかけて、自分の気に入る見た目を探り続けた。
苦心点はやはり、直前の検索結果をなかなか消せなかったことだ。
初めは消えていないことに気づかず、ただ検索結果が変わらないなぁとしか思えていなかった。 そのためすぐに気付けず、修正に遅れたという点。そして消そうと思ったら消えない。新しい検索結果が出てこない。 とさまざまな失敗を重ねた結果今の方法で無事消えたことでやっと安心できた。
index.htmlでの第n回の記述のところの、端にいくにつれているが薄くなっていく見た目は、初めて作った時に個人的にも綺麗で気に入り、いろいろなところで利用している。自分の作風として使ったので、ここを強調したい。
今テーマの感想としては、webページの作り方を学習でき、これから簡易的なものであれば作れるようになったと思える。もちろんこれを仕事にしている人と比べれば、まだまだクオリティーは低く、また、学んでいないことも多くあるはず。 そのため、将来的にwebページを作る可能性があることから、自分でも調べて、新しい知識を得ることで、作れるwebページの幅が広がる。もっと簡単に分かりやすくプログラムを組めるはずなので、このテーマが終わった後も、新たな知識の習得が重要だと考えた。
そして、知識を仕入れても、実際に試さなければ身につかないことも多いだろうから、何かしらの機会に、自分でオリジナルのwebページを作成することもアリではないかと考えた。
難しかったり苦戦したりで大変なことも多かったが、「わかるようになったら楽しい」というのが、今テーマの特徴だと感じた。実際に自分も理解し、自分の想定通りに動いた時の達成感や楽しさは今までの授業の中でもかなり高く、満足できた。
